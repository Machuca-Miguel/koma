KOMA — Design Brief for Stitch AI
===================================
App: Koma — Personal Comic Collection Manager
Platform: Web-first, responsive (also mobile web)
Navigation: Sidebar on desktop / Bottom navigation bar on mobile
Style: Professional, clean, organized. No gradients, no heavy illustration.
       Flat components, clear hierarchy, data-forward.
Font: Geist (sans-serif), system fallback
Radius: 10px base (cards, inputs, buttons share consistent border radius)

  - App logo + name "Koma" at top
  - Navigation items (icon + label): Dashboard, Library, Discover, Collections, Search, Settings

  - Active tab: teal icon + teal label

Layout: Full-width content area. Default grouped by series.

Header row:
  - Page title "Library" (left)
  - View toggle: "Series" | "Comics" (segmented control or pills, teal active)
  - Sort dropdown: "Sort by Series / Title / Year / Date Added / Rating"
  - Search bar (inline, short, magnifier icon)

Series view (default):
  Grid of series cards (3–4 cols desktop, 2 cols tablet, 1 col mobile)
  Each series card:
    - Cover image (top, aspect 2:3 approx.)
    - Series title (bold, 1 line truncated)
    - Publisher (muted text, small)
    - Progress bar: "8 / 12 issues" — teal fill
    - Completion badge: green "Complete" or coral "Ongoing"
    - Clickable → expands to series issue list

Comics view (flat list):
  Same grid but each card shows individual comic:
    - Cover image
    - Title
    - Issue number badge (top-right corner of cover, teal pill)
    - Status badges row: small pills — "Owned" (teal), "Read" (muted), "Wishlist" (coral)
    - Rating: small stars (1–5) below title

Filter chips row (below header):
  Horizontal scrollable chips: "All" | "Owned" | "Read" | "Wishlist" | "Favorites" | "Loaned"
  Active chip: teal background, dark text
  Inactive chip: muted background, secondary text

Empty state:
  Centered icon (open box), "Your library is empty", "Search for comics to add them" (muted text),
  "Search comics" CTA button (teal).

VIEW 5 — SEARCH PAGE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Purpose: Discover and search external databases (GCD, ISBNdb, OpenLibrary) to add comics.
Default search mode: by Series.

Header:
  - Page title "Search"
  - Large search bar centered: placeholder "Search series, titles, ISBN…"
  - Search type toggle pills: "Series" | "Comics" | "ISBN"

Results:
  List layout (not grid) — each result row:
    - Cover thumbnail (56×80px, border-radius 6px)
    - Series/title (bold)
    - Publisher + year range (muted)
    - Issue count badge (small, muted pill)
    - "View issues" button (outlined teal, right-aligned)

On "View issues" click → expands an inline panel or opens a sheet:
  List of issues: number, title, year, cover thumbnail
  Each issue row has "+ Add to library" button (teal icon button)

When an issue is already in the library:
  Show "In library" badge (teal, check icon) instead of add button.

Empty / initial state:
  Centered icon, "Search for comics or series" prompt, muted text.

  - "Synopsis" tab | "My notes" tab

Loan section (card, only shown if comic is loaned):
  - Orange/coral banner: "Loaned to [Name]"
  - Date loaned
  - "Mark as returned" button

SHARED UI PATTERNS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Toast notifications (top-right desktop / bottom center mobile):
  - Success: green-tinted background, check icon, e.g. "Added to library"
  - Error: red-tinted (#F43F5E), X icon
  - Auto-dismiss after 3s, manual dismiss X button

Side Sheet (right slide-in panel, ~400px desktop / full-width mobile):
  Used for: "Add to library", "Edit comic", "Add to collection", "New collection"
  Structure:
    - Header (title + close X button, border-bottom)
    - Scrollable body (form fields)
    - Sticky footer (Cancel + Save/Confirm buttons)
  Backdrop: semi-transparent overlay on desktop

Confirmation dialog:
  Centered modal, max-width 400px
  - Title (bold), description (muted), [Cancel] outlined + [Confirm] teal or [Delete] red
  - Backdrop overlay

Status badges (reused across Library and Comic Detail):
  - Owned:    teal (#00B8A7) small pill
  - Read:     muted gray pill
  - Wishlist: coral (#FF8A65) small pill
  - Favorite: amber/yellow star icon
  - Loaned:   coral pill with user icon

Loading skeleton:
